# Contributing

## Workflow

1. Create an issue or RFC proposal.
2. Branch from `main` using `feature/`, `docs/`, `test/`, or `fix/`.
3. Keep each change focused.
4. Add or update tests for normative behavior.
5. Open a pull request using the repository template.

## Commit convention

Use Conventional Commits:

- `feat:` new behavior
- `fix:` correction
- `docs:` documentation
- `test:` tests or benchmarks
- `refactor:` internal restructuring
- `ci:` continuous integration
- `chore:` maintenance

## RFC changes

A normative requirement MUST be testable. A proposed rule SHOULD include at least one positive example, one counter-example, and one edge case.
