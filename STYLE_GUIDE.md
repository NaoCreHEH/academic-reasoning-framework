# Style Guide

## Normative language

Use MUST, MUST NOT, SHOULD, SHOULD NOT, and MAY for testable requirements.

## RFC structure

Every RFC MUST contain:

1. Status
2. Abstract
3. Motivation
4. Scope
5. Definitions
6. Normative requirements
7. Examples
8. Counter-examples
9. Edge cases
10. Conformance
11. Security and misuse considerations
12. Open questions

## Writing rules

- Prefer observable statements over vague judgments.
- Separate facts, inferences, and hypotheses.
- Define technical terms before using them normatively.
- Avoid uncalibrated numeric confidence claims.
- State when more than one solution can be valid.
