# Security Policy

## Reporting

Do not disclose security vulnerabilities publicly before maintainers have had a reasonable opportunity to investigate.

Open a private security advisory in GitHub when available. If that is not possible, contact a maintainer privately.

## Scope

Security reports may concern:

- unsafe execution guidance;
- prompt injection or tool misuse;
- data leakage;
- benchmark poisoning;
- malicious implementation adapters;
- dependency or workflow vulnerabilities.
