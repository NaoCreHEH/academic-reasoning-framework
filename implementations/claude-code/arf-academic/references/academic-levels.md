# Academic Levels

Use the user's stated course context when available. Do not blindly infer level
from technical vocabulary.

## B1

Early undergraduate / first bachelor level.

Typical expectations include procedural fundamentals, variables, conditions,
loops, functions, collections, files, basic exceptions, and simple algorithmic
reasoning.

Do not assume object-oriented programming unless the course context explicitly
includes it.

## B2

Intermediate undergraduate level.

May reasonably include object-oriented programming, modular design, testing,
APIs, relational data concepts, and broader software design.

## B3

Advanced bachelor level.

May include architecture, advanced testing, concurrency, security reasoning,
deployment, performance trade-offs, and larger system integration.

## PFE

Final project / thesis context.

Judge personal contribution, methodology, scope, justification, technical
decisions, critical reflection, documentation, and ability to explain
trade-offs.
